# Plan 15-01: Core Routing, Open Graph Meta Tags & Deep Link Auth

## Objective
Establish the foundational routing and metadata generation required for sharing content, including dynamic Open Graph tags, an invisible redirect route for instant song playback, and stateful deep-linking through the Google OAuth flow.

## Tasks
1. **Server-Side OG Meta Generation for Existing Routes**
   - **Files:** `frontend/app/artist/[id]/page.tsx`, `frontend/app/album/[id]/page.tsx`, `frontend/app/playlist/[id]/page.tsx`
   - **Action:** Add `generateMetadata` exports to these routes. Ensure they fetch minimal required data (if not already fetched for the page) and return the specified OG tags using the `/api/proxy/image` backend proxy for all images.
     - Artist: `{name} on ഴ-zha` | "Listen to {name}'s music"
     - Album: `{title} by {artist}` | "{count} songs · {year}"
     - Playlist: `{name}` | "{count} songs · Created by {owner}"

2. **Invisible Song Route & Redirect**
   - **Files:** `frontend/app/song/[videoId]/page.tsx`, `frontend/app/song/[videoId]/ClientPlayerRedirect.tsx`
   - **Action:** Create the new server route. `generateMetadata` should fetch track info and output `{title} by {artist}` | "Listen on ഴ-zha". The page body renders `ClientPlayerRedirect`. This client component calls `usePlayerStore.getState().playTrack(videoId)` and `router.replace('/')`. Handle fetch failures by redirecting to `/` with a "Couldn't load shared song" toast.

3. **Stateful Auth Redirects**
   - **Files:** `backend/app/api/auth.py`
   - **Action:** Modify the OAuth initiation route to accept a `return_to` query parameter. Store it in a 10-minute HTTP-only cookie. Sanitize it to prevent open redirects (only allow relative paths starting with `/`). On callback completion, retrieve, delete the cookie, and issue a redirect to the `return_to` path (or `/` if not present).
