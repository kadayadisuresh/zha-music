# Plan 15-02: Share UI Components & Native Share Integration

## Objective
Implement the user interface for sharing content across different devices, utilizing native Web Share APIs on mobile and providing a robust clipboard/QR code fallback on desktop.

## Tasks
1. **Unified Share Utility & Component**
   - **Files:** `frontend/lib/utils/share.ts`, `frontend/components/shared/SharePopover.tsx`, `package.json`
   - **Action:** 
     - Install `qrcode` npm package.
     - Create a utility function that attempts `navigator.share()`. If unsuccessful or unavailable, it falls back to copying to the clipboard.
     - Build a `SharePopover` (for desktop) that displays a "Copy Link" button and a generated QR code representing the URL. On mobile, the button should immediately invoke the native share sheet.

2. **Integrate Share UI into Application**
   - **Files:** 
     - `frontend/components/shared/TrackContextMenu.tsx`
     - `frontend/components/player/PlayerControls.tsx` (or `FullPlayer.tsx` / `NowPlaying.tsx`)
     - `frontend/app/artist/[id]/page.tsx` (header)
     - `frontend/app/album/[id]/page.tsx` (header)
     - `frontend/app/playlist/[id]/page.tsx` (header)
   - **Action:** Add the "Share" action to these existing contexts. Ensure the correct URL is passed (e.g., `${window.location.origin}/song/${track.id}`).

3. **Verify Open Graph Images**
   - **Files:** None
   - **Action:** Perform a manual or automated check to confirm that images referenced in `og:image` tags properly route through the backend proxy (`/api/proxy/image?url=...`) to avoid hotlinking issues.
