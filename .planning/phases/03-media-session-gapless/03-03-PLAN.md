  <name>Task 2: Implement Adaptive Pre-fetch Logic</name>
  <files>frontend/lib/audio/AudioEngine.ts</files>
  <action>
    Inside the monitor loop, implement the trigger for `prepareNext()`.
    Logic:
    1. Get the current connection's `effectiveType` from `navigator.connection` (if available).
    2. Set `prefetchWindow` to 30 (seconds) if type is '2g' or 'slow-2g', otherwise 10 (seconds).
    3. If `activePlayer.duration - activePlayer.currentTime <= prefetchWindow` AND pre-fetch hasn't been triggered yet for the current track AND a `nextTrack` is available in the store:
       - Call `prepareNext(nextTrack)`.
    This implements "Adaptive Pre-fetch" (GAD 1).
  </action>
