<task type="auto">
  <name>Task 1: Create MediaSessionManager</name>
  <files>frontend/lib/audio/MediaSessionManager.ts</files>
  <action>
    Create a `MediaSessionManager` class to encapsulate all `navigator.mediaSession` interactions.
    Include methods to:
    1. `updateMetadata(track: Track)`: Sets `navigator.mediaSession.metadata` with title, artist, and artwork (using high-res thumbnails if possible).
    2. `updatePlaybackState(state: 'playing' | 'paused')`: Sets `navigator.mediaSession.playbackState`.
    3. `updatePositionState(duration: number, currentTime: number)`: Sets `navigator.mediaSession.setPositionState`.
    4. `setActionHandlers(callbacks: { play, pause, next, prev, seek })`: Sets action handlers for 'play', 'pause', 'nexttrack', 'previoustrack', and 'seekto'.
    Handle cases where `navigator.mediaSession` is not available (SSR or older browsers).
  </action>
  <verify>
    <automated>grep -q "navigator.mediaSession" frontend/lib/audio/MediaSessionManager.ts</automated>
  </verify>
  <done>MediaSessionManager provides a clean wrapper for OS media integration.</done>
</task>

<task type="auto">
  <name>Task 2: Integrate MediaSessionManager with AudioEngine</name>
  <files>frontend/lib/audio/AudioEngine.ts</files>
  <action>
    Instantiate `MediaSessionManager` in `AudioEngine`.
    In `AudioEngine.setupEventListeners`, call `updatePlaybackState` and `updatePositionState` based on player events.
    When a new track starts playing, call `updateMetadata`.
    Use `setActionHandlers` to wire OS media buttons to `AudioEngine` methods:
    - 'play' -> `audioEngine.play()` (resume)
    - 'pause' -> `audioEngine.pause()`
    - 'nexttrack' -> Call next track logic (for now, can be a console log if queue logic is missing, or use a mock queue)
    - 'previoustrack' -> Call previous track logic
    - 'seekto' -> `audioEngine.seek(details.seekTime)`
  </action>
